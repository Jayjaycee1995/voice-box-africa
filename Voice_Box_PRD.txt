PRODUCT REQUIREMENTS DOCUMENT (PRD) Project Name: Voice Box Product
Type: African Voice-Over Marketplace (Gig Posting + Bidding + Direct
Invitations)

1.  Product Vision & Objective

Vision: Voice Box is a two-sided marketplace where clients post
voice-over projects and African voice artists submit proposals or
receive direct invitations.

Objectives: - Enable clients to post structured voice-over gigs - Allow
VOs to bid on projects - Allow clients to directly invite specific VOs -
Secure payments with escrow via Paystack - Build a scalable
African-focused voice talent ecosystem

2.  User Roles

Client: - Posts gigs - Invites voice talents - Reviews proposals - Hires
talent and approves delivery

Voice Talent (VO): - Creates profile and uploads demos - Browses gigs -
Submits proposals - Receives direct invitations - Delivers voice files

Admin: - Moderates platform - Manages disputes and payouts - Controls
platform rules

3.  Technical Architecture

Frontend: - React Web App - Mobile-first responsive UI

Backend: - Laravel (API-first) - Laravel Sanctum for authentication -
Laravel Queues for notifications and processing

Database: - MySQL or PostgreSQL

Storage: - AWS S3 or Cloudinary

Payments: - Paystack only (Escrow model)

4.  Core Features

4.1 Landing Page & Talent Search Filters: - Language - Accent - Tone -
Gender - Price Range - Availability

Talent Card: - Photo - Name - Price per word - Rating - Demo play
button - Invite to Project button

4.2 Voice Talent Profile Page - Bio and portfolio - Languages, accents,
tones - Multiple demo audio files with waveform player - Pricing: price
per word and optional flat-rate packages - Availability toggle

4.3 Gig Posting System Gig Fields: - Title - Description - Script upload
or paste - Language, accent, tone - Budget - Usage rights - Deadline -
Visibility (public or invite-only)

4.4 Proposal / Bidding System Proposal Fields: - Bid price - Delivery
time - Proposal message - Demo attachment

Workflow: - Submitted - Shortlisted - Rejected - Hired

4.5 Direct Invitation System - Client searches VO profiles - Invite VO
to project - VO accepts or declines - Accepted invitations create a
proposal automatically

4.6 Hiring & Contract Workflow - Client selects proposal - Contract
created - Client pays via Paystack - Escrow activated - VO starts work

4.7 Messaging System - Chat opens after proposal submission or
invitation acceptance - Phone numbers, emails, and URLs blocked using
regex filters

4.8 Checkout & Escrow - Client pays full amount - Funds held in escrow -
VO sees Funds Secured - VO uploads delivery - Client approves - Platform
releases payout minus commission

4.9 Voice Talent Approval (Automatic) Auto-Verification: - Profile
completeness >= 90% - At least one demo uploaded - Valid audio format
and bitrate

Status Flow: - Pending -> Auto-Approved -> Live - Flagged -> Admin
Review

4.10 Admin Panel - View gigs and proposals - Manage contracts and
disputes - Moderate profiles - Set commission and platform rules

4.11 Ratings & Reviews - Only after completed contracts - 1â€“5 stars and
written feedback - Admin moderation

5.  Pricing & Business Rules

Pricing Formula: Base = Bid Price Platform Fee = Base x 10% Total =
Base + Platform Fee

Escrow Rules: - Client approval required - Auto-release after set days
if no dispute

6.  Database Schema (Core Tables)

Users: id, role, name, email, password, avatar_url, created_at

Voice_Talent_Profiles: user_id, bio, languages, accents, tones,
price_per_word, availability_status, auto_verified, rating_avg

Demos: id, voice_talent_id, title, audio_url, category, duration,
bitrate

Gigs: id, client_id, title, description, script_text, budget, language,
accent, tone, deadline, visibility, status

Invitations: id, gig_id, voice_talent_id, status

Proposals: id, gig_id, voice_talent_id, bid_price, delivery_time,
proposal_text, demo_url, status

Contracts: id, gig_id, client_id, voice_talent_id, final_price,
deadline, status

Messages: id, contract_id, sender_id, message, file_url, created_at

Payments: id, contract_id, paystack_ref, amount, escrow_status,
payout_status

Reviews: id, contract_id, client_id, voice_talent_id, rating, comment

7.  Non-Functional Requirements

Performance: - CDN for audio - Laravel Scout with Meilisearch for search

Security: - Laravel Sanctum - Role-based access control - Paystack
webhook verification

Mobile UX: - Mobile-first responsive design - Touch-friendly audio
player

8.  Development Roadmap

Phase 1 (MVP): - Talent profiles and search - Gig posting - Proposal
system - Direct invitations - Messaging - Paystack escrow - Auto talent
verification - Basic admin panel

Phase 2 (Growth): - Advanced search filters - Proposal limits/tokens -
Payout automation - Analytics dashboard

9.  Deployment Architecture

Hosting: - Laravel API on VPS/AWS/Go54 - React frontend on
Vercel/Netlify - S3/Cloudinary for storage

Strategic Insight: This hybrid Upwork + Direct Invite model ensures fast
marketplace liquidity, premium hiring control, and scalable growth for
the African voice talent ecosystem.
